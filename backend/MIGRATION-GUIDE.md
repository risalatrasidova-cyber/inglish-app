# Руководство по миграциям

## ✅ Первая миграция создана

Эта миграция создает все 8 таблиц:
1. lessons
2. words
3. users
4. admins
5. level_progress
6. word_progress
7. rewards
8. admin_logs

## 🚀 Применение миграции

### 1. Убедитесь, что база данных создана

```sql
CREATE DATABASE inglish_app;
```

### 2. Настройте `.env` файл

Скопируйте `.env.example` в `.env` и заполните данные для подключения к БД.

### 3. Примените миграцию

```bash
npm run migration:run
```

## 📝 Создание новых миграций

После изменения entities создайте новую миграцию:

```bash
npm run migration:generate -- src/migrations/AddNewField
```

**Важно:** Замените `AddNewField` на описательное имя миграции.

## 🔄 Откат миграции

Если нужно откатить последнюю миграцию:

```bash
npm run migration:revert
```

## 📊 Просмотр статуса миграций

```bash
npm run migration:show
```

## ⚠️ Важные замечания

1. **НЕ используйте `synchronize: true`** в production
2. **Всегда создавайте миграции** для изменения схемы БД
3. **Проверяйте миграции** перед применением
4. **Делайте бэкап БД** перед применением миграций в production

## 🔍 Проверка после применения

После применения миграции проверьте, что все таблицы созданы:

```sql
SHOW TABLES;
```

Должны быть созданы все 8 таблиц.
