# КАБИНЕТ АДМИНИСТРАТОРА

## 🎯 Общая концепция

Кабинет администратора предназначен для управления контентом приложения: добавление, редактирование и удаление уроков, слов, а также загрузка аудио файлов. Администратор имеет полный доступ к управлению базой данных приложения.

---

## 🔐 Авторизация администратора

### **Вход в кабинет:**
- Отдельная страница входа для администраторов
- **Логин** (обязательно)
- **Пароль** (обязательно) ⭐ **Отличие от пользовательского входа (там только логин)**
- Кнопка "Войти"
- Возможность "Забыли пароль?" (опционально)

### **Безопасность:**
- Пароль должен быть надежным (минимум 8 символов, буквы + цифры)
- Сессия администратора должна иметь ограниченное время жизни (например, 1 час бездействия)
- При выходе из кабинета сессия завершается
- Логирование всех действий администратора (кто, что, когда изменил)

---

## 📱 Главный экран кабинета администратора

### **Навигационное меню (верхняя панель):**

1. **Логотип/Название приложения** (слева)
   - При клике возвращает на главный экран кабинета

2. **Разделы меню:**
   - **"Уроки"** - управление уроками
   - **"Слова"** - управление словами
   - **"Пользователи"** (опционально) - просмотр статистики пользователей
   - **"Статистика"** (опционально) - общая статистика приложения

3. **Иконка администратора** (справа)
   - Показывает логин администратора
   - При клике открывается меню:
     - **"Выход"** - выход из кабинета администратора

### **Основной контент:**
- Дашборд с краткой статистикой:
  - Количество уроков
  - Количество слов
  - Количество пользователей (если доступно)
  - Последние добавленные слова/уроки

## 📚 Управление уроками

### **Список уроков:**

**Отображение:**
- Таблица со списком всех уроков
- Колонки:
  - **ID** (номер урока)
  - **Название урока** (редактируемое)
  - **Количество слов** в уроке
  - **Дата создания**
  - **Дата последнего изменения**
  - **Действия** (кнопки: Редактировать, Удалить)

**Функционал:**
- **Кнопка "Добавить урок"** (вверху списка)
- **Поиск по названию урока** (поисковая строка)
- **Сортировка** (по названию, дате создания, количеству слов)
- **Пагинация** (если уроков много)

### **Добавление/Редактирование урока:**

**Форма:**
- **Название урока** (обязательное поле, текстовое)
  - Пример: "Урок 1: Животные", "Урок 2: Еда", "Урок 3: Цвета"
- **Описание урока** (опционально, текстовое поле)
- **Порядок отображения** (опционально, числовое поле)
  - Определяет порядок уроков на главном экране приложения
- **Активен** (чекбокс)
  - Если неактивен, урок не отображается пользователям

- **Слова урока** ⭐ **НОВОЕ - Множественный выбор**
  
  **Интерфейс выбора слов:**
  - **Область со списком слов** (прокручиваемый список)
  - **Каждое слово отображается с чекбоксом:**
    - Английское слово | Русский перевод [☐]
    - Пример: "Hello | Привет [☐]"
  - **Поиск по словам** (поисковая строка над списком)
    - Поиск работает по английскому слову и русскому переводу
    - Фильтрует список в реальном времени
  - **Кнопки массового выбора:**
    - **"Выбрать все"** - отмечает все слова в списке
    - **"Снять выбор"** - снимает все отметки
    - **"Выбрать отфильтрованные"** - отмечает только слова, видимые после фильтрации
  - **Счетчик выбранных слов:**
    - Отображается над списком: "Выбрано: X из Y слов"
    - Обновляется в реальном времени при выборе/снятии выбора
  - **Пагинация** (если слов много, можно разбить на страницы)
  
  **При создании нового урока:**
  - Все чекбоксы не отмечены
  - Можно выбрать любое количество слов
  - Можно выбрать все слова сразу
  
  **При редактировании существующего урока:**
  - Слова, уже привязанные к уроку, отмечены чекбоксами
  - Можно добавить новые слова (поставить галочку на новых словах)
  - Можно убрать слова из урока (снять галочку с уже выбранных слов)
  - При сохранении:
    - Слова с отмеченными чекбоксами → привязываются к уроку
    - Слова со снятыми чекбоксами → отвязываются от урока
  
  **Обработка граничных случаев:**
  - Если слов еще нет в системе → показывается сообщение "Сначала добавьте слова" с кнопкой "Добавить слово"
  - Если все слова уже привязаны к другим урокам → показывается сообщение "Все слова уже привязаны к урокам"
  - Можно создать урок без слов (но тогда он будет пустым и не будет отображаться в приложении)
  
  **Важно:** 
  - Одно слово может принадлежать только одному уроку (при привязке к новому уроку автоматически отвязывается от старого)
  - Или можно разрешить одному слову принадлежать нескольким урокам? (нужно уточнить)

- **Кнопки:**
  - **"Сохранить"** - сохраняет урок и привязывает выбранные слова
  - **"Отмена"** - возвращает к списку уроков без сохранения

**Валидация:**
- Название урока не может быть пустым
- Название урока должно быть уникальным (или можно разрешить дубликаты?)
- Можно создать урок без слов (но тогда он будет пустым)

## 📝 Управление словами

### **Список слов:**

**Отображение:**
- Таблица со списком всех слов
- Колонки:
  - **ID** (номер слова)
  - **Английское слово**
  - **Русский перевод**
  - **Урок** (название урока, к которому относится слово)
  - **Аудио файл** (название файла или иконка "есть/нет")
  - **Дата создания**
  - **Действия** (кнопки: Редактировать, Удалить)

**Функционал:**
- **Кнопка "Добавить слово"** (вверху списка)
- **Фильтр по уроку** (выпадающий список - выбрать урок для фильтрации)
- **Поиск** (по английскому слову или русскому переводу)
- **Сортировка** (по английскому слову, русскому переводу, уроку, дате)
- **Пагинация** (если слов много)
- **Массовые действия:**
  - Выбрать несколько слов и удалить их
  - Выбрать несколько слов и переместить в другой урок (через выпадающий список "Переместить в урок")
  - Выбрать несколько слов и добавить к уроку (если слово может принадлежать нескольким урокам)

### **Добавление/Редактирование слова:**

**Форма:**

1. **Английское слово** (обязательное поле, текстовое)
   - Ввод только английских букв (a-z, A-Z)
   - Автокапитализация первой буквы (опционально)
   - Пример: "Hello", "Cat", "Dog"

2. **Русский перевод** (обязательное поле, текстовое)
   - Ввод русских букв
   - Пример: "Привет", "Кот", "Собака"

3. **Урок** (обязательное поле, выпадающий список)
   - Выбор урока, к которому относится слово
   - Если уроков нет, показывается сообщение "Сначала создайте урок"
   - **Альтернативный вариант:** Можно добавить слово без урока, а потом привязать к уроку при редактировании урока (через множественный выбор)

4. **Аудио файл** (обязательное поле, загрузка файла)
   - **Кнопка "Выбрать файл"** или **Drag & Drop** (перетаскивание файла)
   - Поддерживаемый формат: **MP3**
   - Максимальный размер файла: **5 МБ** (можно настроить)
   - После загрузки:
     - Показывается название файла
     - Кнопка "Прослушать" (для проверки)
     - Кнопка "Удалить" (для удаления загруженного файла)
   - **Важно:** Аудио должно быть произношением английского слова

5. **Предпросмотр** (опционально):
   - После заполнения всех полей можно прослушать аудио
   - Проверить правильность написания

**Кнопки:**
- **"Сохранить"** - сохраняет слово
- **"Отмена"** - возвращает к списку слов без сохранения

**Валидация:**
- Английское слово не может быть пустым
- Русский перевод не может быть пустым
- Урок должен быть выбран
- Аудио файл должен быть загружен
- Формат аудио файла должен быть MP3
- Размер аудио файла не должен превышать максимальный размер

**Обработка ошибок:**
- Если файл не MP3 → показать ошибку "Поддерживается только формат MP3"
- Если файл слишком большой → показать ошибку "Файл слишком большой (максимум X МБ)"
- Если не все обязательные поля заполнены → показать ошибку "Заполните все обязательные поля"

## 🎤 Загрузка и управление аудио файлами

### **Процесс загрузки:**

1. **Выбор файла:**
   - Через кнопку "Выбрать файл" (стандартный input file)
   - Или через Drag & Drop (перетаскивание файла в область загрузки)

2. **Валидация файла:**
   - Проверка формата (только MP3)
   - Проверка размера (максимум 5 МБ)
   - Показ ошибки, если файл не подходит

3. **Загрузка:**
   - Показ прогресс-бара загрузки
   - После загрузки файл сохраняется на сервере
   - Генерируется уникальное имя файла (например: `word_123_hello.mp3`)
   - Путь к файлу сохраняется в базе данных

4. **Хранение:**
   - Файлы хранятся на сервере в папке `/uploads/audio/` (или аналогичной)
   - Или в облачном хранилище (S3, Cloud Storage и т.д.)

### **Управление загруженными файлами:**

- **Прослушивание:** кнопка "Прослушать" рядом с загруженным файлом
- **Удаление:** кнопка "Удалить" для удаления файла
- **Замена:** можно загрузить новый файл вместо существующего

  - **Последний вход**

При нажатии "Просмотреть статистику" открывается детальная информация:

  - Все вознаграждения пользователя
- **Подтверждение удаления** (модальное окно с предупреждением)

## 📋 Структура данных для кабинета администратора

### **Таблица: Lessons (Уроки)**
- `id` (INT, PRIMARY KEY, AUTO_INCREMENT)
- `name` (VARCHAR, NOT NULL) - название урока
- `description` (TEXT, NULL) - описание урока (опционально)
- `order` (INT, NULL) - порядок отображения
- `is_active` (BOOLEAN, DEFAULT TRUE) - активен ли урок
- `created_at` (timestamp) - дата создания
- `updated_at` (timestamp) - дата последнего изменения

### **Таблица: Words (Слова)**
- `id` (INT, PRIMARY KEY, AUTO_INCREMENT)
- `english_word` (VARCHAR, NOT NULL) - английское слово
- `russian_translation` (VARCHAR, NOT NULL) - русский перевод
- `lesson_id` (INT, FOREIGN KEY -> Lessons.id) - ID урока
- `audio_file_path` (VARCHAR, NOT NULL) - путь к аудио файлу
- `audio_file_name` (VARCHAR, NULL) - оригинальное имя файла
- `created_at` (timestamp) - дата создания
- `updated_at` (timestamp) - дата последнего изменения

### **Таблица: Admins (Администраторы)**
- `id` (INT, PRIMARY KEY, AUTO_INCREMENT)
- `login` (VARCHAR, UNIQUE, NOT NULL) - логин администратора
- `password_hash` (VARCHAR, NOT NULL) - хеш пароля (bcrypt)
- `created_at` (timestamp) - дата создания
- `last_login` (timestamp, NULL) - последний вход

### **Таблица: AdminLogs (Логи действий администратора)**
- `id` (INT, PRIMARY KEY, AUTO_INCREMENT)
- `admin_id` (INT, FOREIGN KEY -> Admins.id) - ID администратора
- `action` (VARCHAR) - действие (create, update, delete)
- `entity_type` (VARCHAR) - тип сущности (lesson, word, user)
- `entity_id` (INT) - ID сущности
- `details` (TEXT, NULL) - детали действия (JSON)
- `created_at` (timestamp) - дата и время действия

## 🔒 Безопасность

### **Меры безопасности:**
1. **Хеширование паролей:** использование bcrypt для хеширования паролей администраторов
2. **Сессии:** использование JWT токенов или сессий для авторизации
3. **CSRF защита:** защита от межсайтовых запросов
4. **Валидация данных:** проверка всех входящих данных на сервере
